????????????????????????????????????????????????????????????????
? CORE CONCEPTS                                                ?
????????????????????????????????????????????????????????????????

VENUE = Physical location (nightclub, bar, lounge)
  - Has operating hours for each day
  - Open on specific weeks of month
  - Owned by venue owner user

BOOKING = DJ performance at venue
  - Specific day of week
  - Specific week of month (1-4)
  - Specific time slot (1 hour)
  - REPEATS EVERY MONTH on same schedule

USER ROLES:
  SysAdmin    ? Full system access
  Manager     ? High-level operations
  Venue Owner ? Manages own venues
  DJ          ? Creates bookings
  User        ? Basic access

????????????????????????????????????????????????????????????????

MUST HAVE:
? Venue Name
? Owner Username
? At least ONE open day with hours
? At least ONE active week selected

PER-DAY SCHEDULE:
  Day         Start Time    Finish Time
  ?????????   ??????????    ???????????
  Friday      18:00         02:00       ? Can be next day!
  Saturday    18:00         04:00

ACTIVE WEEKS:
  ? Week 1 (1st occurrence of each day)
  ? Week 2 (2nd occurrence)
  ? Week 3 (3rd occurrence)  
  ? Week 4 (4th occurrence)

EXAMPLE:
  Open: Friday, Saturday
  Active Weeks: 1, 3
  Result: Venue open 1st & 3rd Fri/Sat of every month

????????????????????????????????????????????????????????????????
? TIME FORMATS & EXAMPLES                                      ?
????????????????????????????????????????????????????????????????

24-HOUR FORMAT (HH:mm)
??????????????????????
00:00 = Midnight
06:00 = 6 AM
12:00 = Noon
18:00 = 6 PM
22:00 = 10 PM
23:59 = 11:59 PM

OVERNIGHT HOURS:
Start: 22:00 (10 PM Friday)
Finish: 04:00 (4 AM Saturday)
Creates slots: 22:00, 23:00, 00:00, 01:00, 02:00, 03:00


????????????????????????????????????????????????????????????????
? WEEK NUMBER CALCULATION                                      ?
????????????????????????????????????????????????????????????????

Week 1 = Days 1-7 of month
Week 2 = Days 8-14 of month  
Week 3 = Days 15-21 of month
Week 4 = Days 22-31 of month

EXAMPLE - January 2024:
  Jan 6 (Sat) = Week 1
  Jan 13 (Sat) = Week 2
  Jan 20 (Sat) = Week 3
  Jan 27 (Sat) = Week 4

? BOOKING RULES                                                ?
????????????????????????????????????????????????????????????????

? ALLOWED:
  - Multiple DJs, same venue, different time slots
  - Same DJ, multiple venues, same time
  - Same DJ, same venue, different weeks
  - Same DJ, same venue, different days

? NOT ALLOWED:
  - Two DJs, same venue, same day/week/time
  - Booking outside venue's operating hours
  - Booking on day venue is closed
  - Booking on week venue is inactive

? QUICK WORKFLOWS                                              ?
????????????????????????????????????????????????????????????????

CREATE VENUE:
  1. Main Menu ? Venues
  2. Click "Register New Venue"
  3. Enter name & description
  4. Add days with start/finish times
  5. Select active weeks
  6. Save

CREATE BOOKING:
  1. Main Menu ? Bookings
  2. Click "Create Booking"
  3. Select DJ and venue
  4. Choose day, week, time slot
  5. Enter/verify streaming link
  6. Set status ? Save

CREATE DJ USER:
  1. Main Menu ? Users
  2. Click "Create New User"
  3. Enter username, password, details
  4. Set Role: User
  5. Check "Is DJ" box
  6. Enter streaming link ? Save

? TROUBLESHOOTING                                              ?
????????????????????????????????????????????????????????????????

PROBLEM: Can't create booking
FIX: 
  - Ensure DJ account exists
  - Verify DJ flag is checked
  - Confirm venue is active
  - Check time slot is available

PROBLEM: Venue not showing time slots
FIX:
  - Edit venue
  - Add days with start/finish times  
  - Select at least one active week
  - Save changes

PROBLEM: Booking shows conflict
FIX:
  - Choose different time slot
  - OR different week number
  - OR different day

PROBLEM: Can't see streaming link
FIX:
  - Must be authorized user:
    • SysAdmin
    • Manager
    • Venue Owner
    • DJ (own booking only)

? DATA VALIDATION                                              ?
????????????????????????????????????????????????????????????????

VENUE:
  Name: 1-100 characters
  Start Time: HH:mm format (00:00-23:59)
  Finish Time: HH:mm format (00:00-24:00, can be next day)
  Active Weeks: At least one selected
  Open Days: At least one configured

BOOKING:
  DJ Name: 1-50 characters
  Time Slot: Must be from venue's available slots
  Week: 1, 2, 3, or 4 only
  Day: Monday through Sunday
  No double-booking same slot

USER:
  Username: 3-20 chars, alphanumeric + underscore
  Password: Minimum 6 characters
  Email: Valid format
  Role: Valid enum value
  Username must be unique
  Email must be unique
