# Dropdown Menu Spacing Fix

## Problem
Dropdown menu items were too close together with no visual separation. Adding simple margins would create gaps where the mouse hover would cause the menu to close unexpectedly.

## Solution
Implemented a **two-border technique** to create transparent, hit-testable spacing between menu items:

### How It Works

```
???????????????????????????????????
?  Transparent Outer Border       ? ? Captures mouse events (keeps menu open)
?  ?????????????????????????????  ?
?  ? Gray Button (#7D7D7D)    ?  ? ? Visible menu item
?  ?????????????????????????????  ?
?  Transparent Gap (2px padding)  ? ? Visual separation
???????????????????????????????????
???????????????????????????????????
?  Transparent Outer Border       ? ? Mouse still "inside" menu
?  ?????????????????????????????  ?
?  ? Next Gray Button         ?  ?
?  ?????????????????????????????  ?
???????????????????????????????????
```

## Technical Implementation

### SubMenuItemStyle Changes

**Outer Border**:
- `Background="Transparent"` - Invisible but captures mouse events
- `Padding="0,2"` - Creates 2px vertical spacing (top & bottom)
- Keeps menu open when hovering over gaps

**Inner Border**:
- `Background="#7D7D7D"` - The visible gray button
- `Padding="10,5"` - Button content padding
- Only highlights on direct hover

**Result**:
- ? Visual gaps between menu items
- ? Menu stays open when hovering over gaps
- ? Smooth user experience
- ? Transparent spacing as requested

## Technical Notes
The key is the **OuterBorder** with `Background="Transparent"`:
- Transparent backgrounds in WPF **are hit-testable** (capture mouse events)
- This keeps the menu open even when hovering over spacing
- Only `Background="null"` or no background would be non-hit-testable

This is different from CSS where transparent elements might not capture events!
