## üìã PROJECT OVERVIEW

**Goal:** Create a single-file HTML/CSS/JavaScript web application that replicates ALL features of the .NET desktop application, connecting to the existing Azure Cosmos DB backend.

**Target Hosting:** Neocities or similar static hosting

**Database:** Azure Cosmos DB (existing backend)
- Endpoint: `https://fallen-collective.documents.azure.com:443/`
- Collections: users, bookings, venues, chatMessages, userReports, moderationActions, supportMessages, userChatSettings

---

## ‚ö†Ô∏è CRITICAL SECURITY CONSIDERATION

**Problem:** Cosmos DB connection keys cannot be safely embedded in client-side JavaScript - they would be visible to anyone viewing the page source.

**Solutions (Choose One):**

### Option 1: Azure Functions API Layer (RECOMMENDED - SECURE)
- Create serverless Azure Functions as a secure API
- Functions connect to Cosmos DB with keys on backend
- HTML app calls Functions via HTTPS
- **Pros:** Secure, scalable, production-ready
- **Cons:** Requires additional Azure setup

### Option 2: Cosmos DB REST API with Read-Only Key (PARTIAL SECURITY)
- Use Cosmos DB REST API directly from JavaScript
- Use read-only resource tokens with limited permissions
- **Pros:** Simpler setup
- **Cons:** Still exposes keys, limited write capabilities

### Option 3: Accept Risk (NOT RECOMMENDED - INSECURE)
- Embed full access keys in JavaScript
- **Pros:** Quick development
- **Cons:** Anyone can access/modify database

**RECOMMENDATION:** Start with Option 3 for rapid development, then migrate to Option 1 for production.

## üìù NOTES FOR NEXT DEVELOPER

1. **Cosmos DB Connection:** Current plan uses direct connection for rapid development. For production, migrate to Azure Functions API.

2. **RadioBoss API:** Uses public API with key. No sensitive data exposed.

3. **PostImages Upload:** Uses public API for DJ logo hosting. Consider alternatives if rate-limited.

4. **Real-time Chat:** Currently using polling (5-second intervals). Consider SignalR or WebSockets for better performance.

5. **Theme System:** All 7 themes are defined in JavaScript. Custom theme colors save to user profile.

6. **Permissions:** Enforced client-side only. For production, add server-side validation in Azure Functions.
